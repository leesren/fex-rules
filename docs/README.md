# 最佳原则

> 无规矩不成方圆 无规范不能协作。

> 坚持制定好的代码规范，无论团队人数多少，代码应该同出一门。

> 遵守代码规范，你好，我好，大家好。

## Why?
坚持组织应用的结构目的，达到这些目的：
- `快速定位 (Locate) 代码`
- `一眼识别 (Identify) 代码`
- `尽量保持扁平结构 (Flattest)` 
- `遵循 DRY (Do Not Repeat Yourself, 不重复自己) 原则`


坚持四项基本原则定义文件结构，上面的原则是按重要顺序排列的。

为何？LIFT 提供了一致的结构，它具有扩展性强、模块化的特性。因为容易快速锁定代码，提高了开发者的效率。 另外，检查应用结构是否合理的方法是问问自己：**我能快速打开与此特性有关的所有文件并开始工作吗？**
